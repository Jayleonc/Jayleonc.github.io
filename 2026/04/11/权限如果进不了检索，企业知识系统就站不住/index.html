权限如果进不了检索，企业知识系统就站不住

2026-04-11 5 分钟阅读 1965 字

企业级 RAG 系统的安全性基石在于权限一致性。本文深入探讨了 Knowledge Hub 如何通过下沉到 SQL 层的实时 ACL Scope 和 VFS 穿透逻辑，构建起坚不可摧的知识安全边界。

AI knowledge-hub 工程实践 权限

在构建企业级 RAG（检索增强生成）系统时，一个不可逾越的核心设计准则是：用户的可见性边界在“内容管理视图”与“AI 检索视图”中必须保持原子级的强一致性。

如果在交互页面上不可见的内容，可以通过 AI 聊天对话被检索出来，说明系统的安全边界存在结构性缺陷。为了守住这条红线，Knowledge Hub (KH) 经历了一次从“应用层过滤”到“实时检索层 RLS（行级安全）”的重大架构演进。

一、为什么“应用层过滤”和“静态元数据”是不充分的？

在早期 RAG 实践中，开发者常采用两种方案：

  1. Prompt 级拦截：在 System Prompt 中加入“请勿回答关于财务的内容”。这完全依赖模型的遵循能力，面对 Prompt 注入攻击时毫无防御力。
  2. 静态元数据过滤：将权限 Token 写入向量数据库的 Metadata。这看似安全，但存在“一致性空窗期”。当一个文件的权限变更时，更新上千万个向量切片的元数据可能需要数秒乃至数分钟，这在金融、军工等敏感领域是不可接受的。

KH 的实现逻辑是：权限校验必须是检索动作的“前置条件”，且必须是实时的。

二、底层进化：基于实时 SQL Scope 的权限下沉

在 KH 的 internal/database/scope/acl.go 中，我们实现了一套极其精密的 ACLScope 机制。它不再依赖预计算的 Token，而是将复杂的权限语义直接转化为检索时的 SQL 片段。

2.1 统一的 ACLScopeForTable

KH 利用 GORM 的 Scope 特性，通过 ACLScopeForTable 函数为所有数据库查询（包括向量检索时的 Segment 查询）强制注入 ACL 约束。

这个函数会动态构建出包含 jsonb_array_elements_text 的 SQL 条件，实时在 document 表的 access_policy（JSONB 字段）中匹配当前的 PrincipalID（用户）、RoleIDs（角色）、Groups（用户组）和 Departments（部门）。

这意味着：无论查询的是文档列表还是向量分片，它们共享同一套权限逻辑。 只要 document 表里的 access_policy 发生变更，下一毫秒的检索结果就会立即响应。

三、VFS 目录穿透：安全性与用户体验的平衡

在处理复杂的虚拟文件系统（VFS）时，权限逻辑面临一个棘手的问题：如果一个用户有权访问 /Project/Secret/Report.pdf，但他对父目录 /Secret/ 没有显式权限，他能找到这个文件吗？

KH 在 ACLScopeForVFSNode 中实现了一套 “穿透性可见性” (Directory Penetration) 逻辑：

  1. 文件节点：必须满足其关联 Document 的实时 ACL 校验。
  2. 目录节点：
    • 若目录有显式策略，则按策略执行。
    • 若目录无显式策略，系统会执行一个 EXISTS 子查询：只要该目录下级存在任何一个用户可访问的文件，该目录即对该用户可见。

这种设计既保证了权限的严密性（不可访问的文件绝不露头），又保证了导航的连贯性，解决了企业级知识库在深度层级下的访问障碍。

四、Fail-Safe：默认拒绝原则

在 KH 的安全性哲学中，没有任何“模糊地带”。

ACLScopeFromSpace 函数的设计体现了极致的 Fail-Safe（安全失败） 原则：如果检索请求中没有提供有效的 Space 上下文（例如 Token 校验失败或上下文丢失），该函数会直接返回 1 = 0 的 SQL 片段。

这意味着，一旦系统检测到安全性参数异常，检索结果将强制为空，而不是默认公开。这种“宁可错杀，绝不漏放”的逻辑，是系统承载核心业务数据的底气。

五、所有者单位（Owner Unit）的兜底授权

除了显式的 access_policy，KH 还引入了 buildOwnerUnitCondition 机制。

在很多企业场景下，文档并没有设置复杂的权限规则，而是遵循“谁创建、谁所在部门可见”的朴素原则。KH 的 ACL Scope 会自动识别资源的 owner_unit。如果一份文档没有显式的 ACL 策略，系统会回退到组织架构关联逻辑，允许同部门的成员访问。这种显式策略与隐式授权的有机结合，极大降低了管理员的配置负担。

总结

权限控制不是 RAG 系统的外挂插件，而是系统内核的核心逻辑。

一个可靠的企业级知识平台，必须确保前台展示事实与后台检索事实的高度对齐。通过将复杂的权限语义下沉到 SQL 层，并实现目录穿透与 Fail-Safe 机制，KH 在保证检索性能的同时，构建起了一道坚不可摧的知识安全红线。对于企业用户而言，看得见的安全性，才是真正的可用性。
